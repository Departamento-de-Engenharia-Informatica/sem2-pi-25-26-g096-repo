# Glossary

**Terms, Expressions and Acronyms (TEA) must be organized alphabetically.**

Column "TEA (PT)" is optional. If the team does not intend to use it, then remove it.

(Complete according to the provided example)

| **_TEA_** (EN) | **_TEA_** (PT) | **_Description_** (EN) |
|:---|:---|:---|
| **Actor** | **Ator** | Individual or external entity that interacts with the platform, triggering system behaviour through defined use cases. |
| **Administrator** | **Administrador** | Platform user responsible for managing access privileges, approving or rejecting registration requests, and registering system entities. |
| **Agile** | **Ágil** | Software development approach based on iterative delivery, collaboration and continuous adaptation to change. |
| **Asset** | **Património** | Any property or financial resource owned by a political agent, including real estate, vehicles, shares or other holdings. |
| **Asset Declaration** | **Declaração de Bens** | Formal document submitted by a political agent detailing their wealth and financial interests at a given point in time, used to detect illicit enrichment by comparison across periods. |
| **Authentication** | **Autenticação** | Mechanism by which the system verifies a user's identity through credentials before granting access. |
| **Authorisation** | **Autorização** | Process of determining whether an authenticated user has permission to perform a specific action in the system. |
| **Backlog** | **Backlog** | Prioritised collection of user stories and development tasks to be addressed across project sprints. |
| **Business Participation** | **Participação Empresarial** | Any ownership stake, quota or shareholding held by a political agent in a company or business entity. |
| **Citizen** | **Cidadão** | Registered platform user who can consult public information about political agents and submit complaints regarding perceived lack of transparency. |
| **Complaint** | **Denúncia** | Report submitted by a citizen or journalist to signal potentially irregular or non-transparent behaviour by a political agent. |
| **Conflict of Interest** | **Conflito de Interesses** | Situation in which a political agent's private interests may compromise or appear to compromise their public duties. |
| **Councillor** | **Vereador** | Political agent who is an elected member of a municipal council. |
| **Credential** | **Credencial** | Set of information, typically a username and password, used to authenticate a user in the system. |
| **Data** | **Dado** | Unit of information stored, processed or presented by the system. |
| **Data Persistence** | **Persistência de Dados** | Ability of the system to retain stored information across successive execution sessions, implemented via object serialization. |
| **Data Serialization** | **Serialização de Dados** | Process of converting objects into a storable or transmittable format so that they can be reconstructed in a future session. |
| **Declaration of Interests** | **Declaração de Interesses** | Structured submission by a political agent containing information about income, positions held, assets and business participations. |
| **Declaration Status** | **Estado da Declaração** | Attribute reflecting the current stage of a declaration in its lifecycle, such as submitted, validated or rejected. |
| **Declaration Type** | **Tipo de Declaração** | Classification of a declaration as initial, regular or exceptional, depending on the circumstances of its submission. |
| **Deputy** | **Deputado** | Political agent elected to serve in a legislative assembly. |
| **Disclosure** | **Divulgação** | Act of making financial or professional information publicly available, particularly regarding assets, income and interests of political agents. |
| **Domain Model** | **Modelo de Domínio** | Conceptual diagram representing the main entities of the problem domain, their attributes and the relationships between them. |
| **Ethics Committee** | **Comissão de Ética** | Body responsible for supervising declarations of interests, verifying incompatibilities and investigating potential violations of the code of conduct. |
| **Function** | **Função** | Specific role or responsibility performed by a political agent within an institution, whether public, private or social. |
| **Income** | **Rendimento** | Any financial gain received by a political agent, including salary, side income, subsidies or support from institutions. |
| **Institution** | **Instituição** | Organisation in which a political agent holds or has held a position, classifiable as a company, political party, foundation, institute or association. |
| **Journalist** | **Jornalista** | Platform user registered with the Journalists' Union, granted specific access rights to support investigative work and public scrutiny. |
| **Mayor** | **Presidente de Câmara** | Political agent who leads a municipal government. |
| **Minister** | **Ministro** | Senior political agent responsible for leading a governmental ministry. |
| **Political Agent** | **Agente Político** | Individual exercising a public political function, such as a deputy, minister, councillor, mayor or parish council president. |
| **Position** | **Cargo** | Office or responsibility held by a political agent in a public, private or social institution. |
| **Registration Request** | **Pedido de Registo** | Submission made by a prospective platform user requesting an account with a specific role, pending administrator approval. |
| **Role** | **Papel** | Classification of a platform user that determines their access level and the actions they are permitted to perform. |
| **Submission** | **Submissão** | Act of formally sending a declaration or request to the system for processing or review. |
| **System** | **Sistema** | The software platform comprising the Transparency Portal, responsible for managing declarations, users and analysis tools. |
| **System Administrator** | **Administrador de Sistema** | Person or team responsible for managing the system, its access privileges and user accounts. |
| **Transparency Portal** | **Portal da Transparência** | Main deliverable of the project; an IT platform supporting the registration, validation and analysis of declarations of interests by political agents. |
| **User** | **Utilizador** | Any individual who accesses the platform, identified by credentials and assigned a role that determines their permissions. |
| **User Story** | **História de Utilizador** | Concise description of a system feature written from the perspective of a platform user, expressing a need and its purpose. |
| **Validation** | **Validação** | Process by which an Ethics Committee member reviews a submitted declaration and either approves it or identifies sections requiring correction. |
